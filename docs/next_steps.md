## 1. Интерпретация кластеров совместно с руководителем

### Описание
После получения кластеров необходимо **соотнести каждый кластер
с физически осмысленным типом поверхностной структуры**. Это ключевой этап валидации,
который невозможен без экспертного знания кристаллографии.

### Ожидаемые типы мотивов
| Мотив | Описание | Характерные признаки (n1–n5) |
|-------|----------|------------------------------|
| **Террасы** (terraces) | Плоские участки с полной координацией | Все n_i близки к максимумам (8, 6, 12, 24, 8) |
| **Ступени** (steps / ledges) | Одномерные обрывы между террасами | Промежуточные значения n1–n2, сниженные n3–n5 |
| **Изломы** (kinks) | Точечные дефекты на ступенях | Резко сниженные n1–n3 |
| **Адатомы** (adatoms) | Одиночные выступающие атомы | Минимальные n1 (2–3) |
| **Вершины** (apices) | Пересечение нескольких ступеней | Сниженные n_i во всех порядках |

### Как провести
1. Для каждого кластера вычислить средние значения n1–n5 и стандартные отклонения.
2. Визуализировать типичные патчи каждого кластера (по 16–20 примеров).
3. Совместно с руководителем сопоставить кластеры с известными типами мотивов.
4. При необходимости объединить или разделить кластеры.

### Статус
- Частично реализовано: `visualize_crystal.py` уже содержит `plot_cluster_examples()`.
- `extract_crystal_embeddings.py` вычисляет метрики кластеризации.
- **Не завершено:** встреча с руководителем для интерпретации.

## 2. Оптимизация числа кластеров

### Статус: ✅ РЕАЛИЗОВАНО

Систематический перебор k реализован в `src/crystal/optimize_clusters.py`:
- Диапазон k = 3..20 с метриками Silhouette, CH, DB, Inertia, AMI/NMI.
- Результаты: `data/crystal/analysis/cluster_optimization_metrics.csv`.
- Обоснование выбора k = 8: `data/crystal/analysis/k_selection_justification.md`.

**Открытый вопрос:** научный руководитель считает, что при данном радиусе модели
число зон ближе к ~20. Рекомендуется рассмотреть k ∈ {12, 15, 20} для сравнения.

### Статус: ✅ РЕАЛИЗОВАНО

## 5. Тестирование на других типах решёток (ГЦК, ГПУ)

### Описание
Пайплайн разработан для BCC-решётки, но его архитектура универсальна
и должна работать для других типов кристаллических решёток.

### Целевые решётки
| Решётка | Название | Примеры металлов |
|---------|----------|-------------------|
| **ГЦК (FCC)** | Гранецентрированная кубическая | Au, Ag, Cu, Al, Pt |
| **ГПУ (HCP)** | Гексагональная плотноупакованная | Ti, Zr, Mg, Co |

### Что потребуется адаптировать
1. Данные: получить аналогичные файлы координат для FCC/HCP кристаллов.
2. Генератор патчей: скорректировать MAX_NEIGHBORS нормировочные множители.
3. Модель: архитектура не потребует изменений (5 каналов сохраняются).
4. Обучение: отдельная модель для каждой решётки или transfer learning.

### Статус: Не начато. Требуются данные от руководителя.
