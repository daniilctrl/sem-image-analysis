# Дальнейшие шаги — детальный план развития проекта

## 1. Интерпретация кластеров совместно с руководителем

### Описание
После получения 8 (или иного количества) кластеров необходимо **соотнести каждый кластер
с физически осмысленным типом поверхностной структуры**. Это ключевой этап валидации,
который невозможен без экспертного знания кристаллографии.

### Ожидаемые типы мотивов
| Мотив | Описание | Характерные признаки (n1–n5) |
|-------|----------|------------------------------|
| **Террасы** (terraces) | Плоские участки с полной координацией | Все n_i близки к максимумам (8, 6, 12, 24, 8) |
| **Ступени** (steps / ledges) | Одномерные обрывы между террасами | Промежуточные значения n1–n2, сниженные n3–n5 |
| **Изломы** (kinks) | Точечные дефекты на ступенях | Резко сниженные n1–n3 |
| **Адатомы** (adatoms) | Одиночные выступающие атомы | Минимальные n1 (2–3), очень высокий grayscale |
| **Вершины** (apices) | Пересечение нескольких ступеней | Сниженные n_i во всех порядках |

### Как провести
1. Для каждого кластера вычислить **средние значения n1–n5** и **стандартные отклонения**.
2. Визуализировать типичные патчи каждого кластера (по 16–20 примеров).
3. Совместно с руководителем сопоставить кластеры с известными типами мотивов.
4. При необходимости объединить или разделить кластеры.

### Файлы для реализации
- Добавить в `visualize_crystal.py` функцию `plot_cluster_examples()` для отображения
  типичных патчей каждого кластера.
- Добавить в `extract_crystal_embeddings.py` вывод средних n1–n5 по кластерам.

## 2. Оптимизация числа кластеров (Silhouette, Elbow)

### Описание
Текущее число кластеров (8) было выбрано эвристически. Необходимо **систематически подобрать
оптимальное k** с использованием стандартных метрик.

### Метрики для оценки
| Метрика | Что оценивает | Оптимум |
|---------|--------------|---------|
| **Silhouette Score** | Насколько объекты похожи на свой кластер vs. ближайший чужой | Максимум (ближе к 1) |
| **Calinski-Harabasz Index** | Отношение межкластерной дисперсии к внутрикластерной | Максимум |
| **Davies-Bouldin Index** | Средняя «похожесть» между кластерами | Минимум (ближе к 0) |
| **Elbow Method (инерция)** | Сумма квадратов расстояний внутри кластеров | Точка «перегиба» |

### Как провести
1. Запустить KMeans для k = 3, 4, 5, 6, 7, 8, 10, 12, 15, 20.
2. Для каждого k вычислить все четыре метрики (Silhouette уже реализован).
3. Построить графики зависимости метрик от k.
4. Выбрать k, дающее наилучший баланс метрик.
5. Сравнить с физически ожидаемым числом мотивов (обычно 4–6 типов).

### Файлы для реализации
- Расширить `extract_crystal_embeddings.py`: добавить цикл по диапазону k
  и построение графиков метрик.
- Результаты сохранять в `data/crystal/embeddings/cluster_metrics.csv`.

## 3. Сравнение с аналитической классификацией (индексы Миллера)

### Описание
Для полусферы BCC-кристалла существует **аналитическое** описание поверхности через
**индексы Миллера** {hkl}. Каждой точке на полусфере можно приписать ближайшую
кристаллографическую грань. Это даёт «эталонную» классификацию для сравнения
с нашей ML-кластеризацией.

### Основные грани BCC-кристалла
| Индексы Миллера | Тип поверхности | Расположение на полусфере |
|-----------------|-----------------|--------------------------|
| {110} | Плотноупакованная, максимальная координация | Пояс при ~45° |
| {100} | Менее плотная, квадратная решётка | Полюсы (0,0,1), (1,0,0), (0,1,0) |
| {111} | Открытая, низкая координация | Углы при 54.7° |
| {112}, {123} | Высоко-индексные, ступенчатые | Между основными гранями |

### Как провести
1. Для каждого поверхностного атома вычислить направление нормали: `n̂ = (X, Y, Z) / R`.
2. Определить ближайшую грань {hkl} через угловое расстояние до эталонных направлений.
3. Построить карту индексов Миллера на полусфере (аналог стереографической проекции).
4. Вычислить **матрицу соответствия** между ML-кластерами и гранями Миллера.
5. Оценить точность: какой процент атомов одной грани попал в один кластер.

### Файлы для реализации
- Новый файл: `src/crystal/miller_indices.py` — расчёт ближайших граней.
- Новый файл: `src/crystal/compare_clusters_miller.py` — confusion matrix, метрики.

## 4. Тестирование на других типах решёток (ГЦК, ГПУ)

### Описание
Пайплайн разработан для BCC-решётки, но его архитектура **универсальна**
и должна работать для других типов кристаллических решёток.

### Целевые решётки
| Решётка | Название | Примеры металлов | Особенности поверхности |
|---------|----------|-------------------|------------------------|
| **ГЦК (FCC)** | Гранецентрированная кубическая | Au, Ag, Cu, Al, Pt | Другой набор n_i; {111} — самая плотная грань |
| **ГПУ (HCP)** | Гексагональная плотноупакованная | Ti, Zr, Mg, Co | Гексагональная симметрия; базисная и призматическая грани |

### Что потребуется адаптировать
1. **Данные**: получить от руководителя аналогичные файлы координат для FCC/HCP кристаллов.
2. **Генератор патчей**: скорректировать максимумы нормировки `[n1_max, ..., n5_max]`,
   так как у FCC и HCP другие координационные числа.
3. **Модель**: архитектура ResNet18 и SimCLR не потребуют изменений (входные каналы те же).
4. **Обучение**: можно обучить отдельную модель для каждой решётки, либо одну общую
   (transfer learning с предобученными весами от BCC).
5. **Визуализация**: код визуализации полностью переиспользуется.

### Ожидаемый результат
- Подтверждение универсальности подхода.
- Сравнение качества кластеризации на разных решётках.
- Публикация / доклад с результатами на нескольких системах.
