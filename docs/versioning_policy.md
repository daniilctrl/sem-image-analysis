# Политика версионирования (SEM + Crystal)

## Цель

Держать репозиторий легким и воспроизводимым: в Git хранить код, конфигурации, схемы данных и компактные таблицы; тяжелые бинарные артефакты хранить вне Git.

## Что коммитим

- Код: `src/`, `notebooks/*.py`, инфраструктурные скрипты.
- Документация: `README.md`, `docs/*.md`.
- Небольшие табличные артефакты для воспроизводимости: `.csv` с метриками, индексами и маппингами.
- Шаблоны/примерные файлы отчетов без тяжелых встроенных ресурсов (по возможности).

## Что не коммитим

- Сырые данные и производные массивы: `.tif`, `.tiff`, `.npy`, `.parquet`, `.h5`.
- Визуализации и отчеты как артефакты экспериментов: `.png`, `.jpg`, `.jpeg`, `.webp`, `.html` (автогенерированные), большие `.pdf`.
- Чекпоинты моделей: `.pth`, `.pt`, `.ckpt`.
- Архивы/временные файлы: `.zip`, `.log`, временные скрипты.

## Правила по направлениям

- **SEM:** в Git оставляем код и итоговые CSV-метрики; тяжелые эмбеддинги/картинки/HTML-отчеты исключаем.
- **Crystal:** первая итерация уже завершена; в Git фиксируем код и интерпретируемые таблицы (`cluster_*stats*.csv`, `*crosstab*.csv`), а массивы патчей/эмбеддингов и изображения храним вне Git.

## Практика работы

1. Перед коммитом проверяем `git status`.
2. Если появился новый тип тяжелого артефакта, сразу добавляем его в `.gitignore`.
3. Для воспроизводимости экспериментов сохраняем:
   - параметры запуска,
   - версию кода (commit hash),
   - ссылки/пути на внешний storage с артефактами.
4. Для переносимости среды поддерживаем актуальный `pyproject.toml` как
   источник истины:
   - `[project].dependencies` — core runtime (train + extract + evaluate),
   - `[project.optional-dependencies].viz` — UMAP, seaborn, plotly, gradio,
   - `[project.optional-dependencies].dataprep` — opencv, tifffile, openpyxl, h5py,
   - `[project.optional-dependencies].reports` — markdown, PyMuPDF,
   - `[project.optional-dependencies].dev` — pytest, pytest-cov.
   `requirements.txt` остаётся как удобный shortcut для `pip install -r`,
   эквивалентный core-набору из pyproject.
