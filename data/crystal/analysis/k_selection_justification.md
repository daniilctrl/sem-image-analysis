# Выбор числа кластеров: near-best диапазон k ∈ [35, 50] для R=50

> Обновлено после расширенного sweep до k=60 и сопоставления с аналитической моделью
> [Никифоров, Егоров, Шень, 2009]. Предыдущая рекомендация k=8 отозвана как
> недооценка структуры поверхности.

## Область применимости

Все числовые результаты ниже получены **для одного конкретного радиуса
полусферы R = 50 параметров решётки** (`data/crystal/coordinates_and_neighbours_50.xlsx`).

Важно: статья [Никифоров и др., 2009] формально гарантирует сходимость
модели «тонкой оболочки» + «оборванных связей» для R ∈ [100, 1000]
параметров решётки. **R=50 лежит ниже нижней границы применимости**.
Полученные выводы строго корректны в формулировке:

> «для данного шара (R=50) и данного представления поверхности
> (SimCLR на 5-канальных патчах 32×32, window_radius=10 п.р.) разумный
> диапазон k ∈ [35, 50]»

Универсальность результата по радиусу полусферы пока **не проверена**
и является предметом multi-radius ablation (см. раздел «Открытые вопросы»).

## Методика выбора k

`optimize_clusters.py` выдаёт не один k, а **near-best диапазон** по
совокупности критериев:

1. **Внутренние метрики**: Silhouette, Calinski-Harabasz, Davies-Bouldin.
2. **Внешняя метрика**: AMI / NMI vs аналитическая классификация Миллера
   (`miller_utils.assign_miller_labels`, 9+ семейств с tolerance 6°).
3. **Retrieval-метрика**: `precision@10_miller` (см. `retrieve_crystal.py`).
4. **Физическая интерпретируемость**: средние n1..n5 на кластер
   (`analyze_clusters_physical.py`) и степень чистоты по Миллеру
   (`analyze_miller_indices.py`).

## Итоговая рекомендация

Для R=50, window_radius=10, при текущей архитектуре CrystalSimCLR:

- **k = 35** — основной balanced candidate. Сбалансированная интерпретируемость
  (достаточно крупных кластеров для физического анализа) и чистота по Миллеру.
- **k = 50** — fine-grained candidate. Чуть выше Miller purity, но кластеры мельче;
  полезен для retrieval, сложнее для ручной интерпретации.
- retrieval по `precision@10_miller` у k=35 и k=50 **практически одинаков**
  (bootstrap 95% CI пересекаются; см. `retrieve_crystal.py --bootstrap`).

k = 8 из предыдущей версии документа **отозван**: это систематическая
недооценка, продиктованная желанием выбрать «единственный» оптимум.
Внутренние метрики продолжают улучшаться при k > 20, а AMI vs Miller
растёт монотонно — что согласуется с формулой (5) статьи (см. ниже).

## Сопоставление с аналитической моделью [Никифоров 2009]

Из формулы (5) статьи для радиуса грани на полусферическом эмиттере:

$$g(hkl) = \frac{2}{3}\sqrt{\frac{3ar}{h^2+k^2+l^2}}$$

В безразмерных единицах (a=1, r=50):

$$g(hkl) = \frac{2}{3}\sqrt{\frac{150}{h^2+k^2+l^2}}$$

| Семейство | \|hkl\|² | g (п.р.) |
|-----------|----------|----------|
| {100}     | 1        | 8.16     |
| {110}     | 2        | 5.77     |
| {111}     | 3        | 4.71     |
| {210}     | 5        | 3.65     |
| {211}     | 6        | 3.33     |
| {311}     | 11       | 2.46     |
| {321}     | 14       | 2.18     |
| {411}     | 18       | 1.93     |
| {510}     | 26       | 1.60     |
| {531}     | 35       | 1.38     |
| {711}     | 51       | 1.14     |

**Число семейств с g ≥ 1 (минимум ~10 атомов в грани):** ~25–35.

Это близко к эмпирическому near-best k ∈ [35, 50]. Лёгкое превышение
(k_emp > k_theory) ожидаемо по двум причинам:

1. При R=50 **все g < window_radius = 10**, т.е. ни одна грань не умещается
   в одно окно целиком. Модель неизбежно кластеризует **смешанные состояния
   на стыках граней** (vicinal, ступени, изломы), что добавляет кластеры
   сверх числа чистых семейств {hkl}.
2. R=50 — ниже области применимости статьи [Никифоров 2009]. На R ≥ 100
   ожидается более чистая картина.

Теоретические предсказания по формуле (5) для будущего multi-radius
ablation — в `scripts/predict_k_theory.py`.

## Гипотезы для multi-radius ablation (H_phys vs H_inv vs H_mix)

Предложены три формальные гипотезы, различающиеся поведением `k_best(R)`:

- **H_phys (физика граней)**: `k_emp(R)` растёт ~ линейно с R, согласно
  формуле (5). Согласованность: критерий `corr(k_emp, R) > 0.9`.
  Интерпретация: модель ловит геометрию семейств Миллера.
- **H_inv (инвариантность к масштабу)**: `k_emp(R)` стабилен, `|Δk| < 20%`
  между разными R. Противоречит аналитике. Интерпретация: модель ловит
  чисто локальные мотивы независимо от глобальной кривизны.
- **H_mix (промежуточное)**: рост сублинейный, `0 < ∂k/∂R < k_theory'(R)`.
  Интерпретация: модель ловит смесь граней и локальных мотивов; часть
  кластеров насыщается, часть растёт с R.

Критерий (не)отвержения формализован до старта ablation, чтобы выбор
интерпретации не подгонялся под наблюдения.

## Открытые вопросы

1. **Данные для R ∈ {100, 200, 316}** — запрошены у руководителя.
   Без них multi-radius ablation невозможен (subset текущего R=50
   даст атомы с неполными neighbor counts, что некорректно).
2. **Цель кластеризации**: «локальные surface motifs» vs «зональная
   сегментация полусферы» — фиксируется H_phys/H_inv/H_mix.
3. **Расширение Miller-семейств**: `miller_utils.py` теперь содержит
   ~28 семейств (было 9) для честной оценки AMI на больших R.
   См. `src/crystal/miller_utils.py`, ссылка на рис. 3 статьи.
